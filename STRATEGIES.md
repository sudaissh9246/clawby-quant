# 策略库 — Binance 日内量化 Bot(v1,10 个策略)

> 硬约束(来自交易习惯):**任何持仓最长 24 小时**,到时强制平仓;所有策略皆为日内完成。
> 因子代号引用 `FACTORS.md`;每个策略默认**关闭**,在 `strategies.yaml` 里改 `enabled: true` 启用。
> 阈值均为起始默认值(标 ⚙ 可调),实盘前建议先用小仓位跑 1-2 周校准。

## 全局风控层(凌驾于所有策略之上)

| 规则 | 逻辑 | 因子 |
|---|---|---|
| 事件静默 | `ECON_CAL` 中 CPI/FOMC/非农公布前 60 分钟 ⚙:禁止开新仓,存量仓位杠杆减半(S09 除外,它专做事件) | `ECON_CAL` |
| 情绪水位 | `FEAR_GREED` ≥ 85(极贪)时多头仓位系数 ×0.6,≤ 15(极恐)时空头仓位系数 ×0.6 ⚙ | `FEAR_GREED` |
| 单币上限 | 同一币种所有策略合计敞口 ≤ 账户净值 10% ⚙ |
| 总敞口上限 | 全部持仓名义价值 ≤ 净值 × 3 ⚙(含杠杆) |
| 日内熔断 | 当日已实现亏损达净值 -3% ⚙:全平并停止开仓至次日 UTC 0 点 |
| 持仓时钟 | 每笔仓位带 `opened_at`,达到策略 `max_hold`(全部 ≤ 24h)无条件市价平仓 |

## S01 · FUNDING_FADE — 极端费率反转

**类型**:均值回归 | **方向**:双向 | **适用**:震荡市与趋势末端

**逻辑**:资金费率是杠杆拥挤度的价格。当聚合费率打到极端且散户同向扎堆,费率结算的经济压力 + 拥挤反向清洗大概率造成向均值的回摆。

- **入场**(15m 扫描):`FUND_AGG_OIW` 1h 费率 ≥ +0.05% ⚙(年化 ≈ +55%)且 `LSR_GLOBAL` ≥ 2.2 ⚙ → 开空;费率 ≤ -0.03% 且 LSR ≤ 0.75 → 开多。用 `FUND_XSEC` 横截面找费率最极端的前 3 个币,只做流动性 Top50 标的。
- **出场**:费率回落至 ±0.01% 内,或浮盈 ≥ 1.5×ATR(`TECH_ATR_XSEC`),或触发止损 1×ATR ⚙。
- **max_hold**:16h(覆盖两个费率结算点)
- **因子**:`FUND_AGG_OIW` `FUND_XSEC` `LSR_GLOBAL` `TECH_ATR_XSEC`
- **风险点**:强趋势中费率可以长期极端(2024 式逼空),必须带 ATR 止损,禁止逆势加仓。

## S02 · LIQ_REBOUND — 爆仓瀑布接针

**类型**:均值回归 | **方向**:只多(可镜像做空版)| **适用**:急跌后的流动性真空

**逻辑**:密集多头爆仓是被迫卖出,卖压来自清算引擎而非新增看空共识;爆仓峰值后价格往往超调,清洗完成后快速回填。

- **入场**(5m 扫描):`LIQ_AGG` 1h 多头爆仓额 ≥ 近 7 日小时均值 6 倍 ⚙,且 30m 跌幅 ≥ 2.5×ATR,且 `LIQ_ORDERS` 大额爆仓单流在最近 10 分钟内衰减 ≥ 70%(瀑布尾声确认),且 `OB_DEPTH` 买盘深度开始回补(bid/ask ≥ 1.1)→ 市价开多。
- **出场**:反弹至跌幅 0.5 回撤位分批止盈;止损为入场后再创新低 0.6×ATR ⚙。
- **max_hold**:4h(接针不隔夜等待)
- **因子**:`LIQ_AGG` `LIQ_ORDERS` `PX_FUT_KLINE` `OB_DEPTH` `TECH_MA_EMA_BOLL_MACD_ATR`
- **风险点**:接飞刀策略,严禁在爆仓仍在放大时入场(必须等衰减确认);黑天鹅(交易所出事)时直接熔断不接。

## S03 · SQUEEZE_RIDE — 轧空/轧多动量

**类型**:动量突破 | **方向**:双向 | **适用**:单边启动初期

**逻辑**:与 S02 互补——当爆仓发生在**反方向**(空头爆仓+价格上破),清算是燃料而非尾声,顺势吃清算链式反应,目标位直接用清算地图的密集区。

- **入场**(5m 扫描):价格突破 4h 区间高点,且 `LIQ_AGG` 空头爆仓开始放量(≥3 倍均值),且 `FUND_AGG_OIW` 费率为负或中性(空头仍未投降),且 `TAKER_FLOW` 买卖比 ≥ 1.4 ⚙ → 追多。镜像条件追空。
- **出场**:目标位 = `LIQ_MAP` 上方最近的清算密集带;移动止损 1.2×ATR 跟随;`TAKER_FLOW` 转向(<0.9)提前离场。
- **max_hold**:8h
- **因子**:`LIQ_AGG` `LIQ_MAP` `FUND_AGG_OIW` `TAKER_FLOW` `PX_FUT_KLINE`
- **风险点**:假突破;要求突破 K 线成交量 ≥ 20 周期均量 2 倍,否则不进。

## S04 · SMART_DUMB_DIV — 大户散户背离跟单

**类型**:仓位情绪 | **方向**:双向 | **适用**:方向酝酿期

**逻辑**:`LSR_TOP_POS` 是大户真金白银的持仓倾斜,`LSR_GLOBAL` 是散户人数情绪;两者显著背离时,历史上大户胜率占优。

- **入场**(15m 扫描):z-score 化两个序列(近 7 日窗口),当 top_pos_z - global_z ≥ +1.5 ⚙(大户显著更多)且价格站上 1h EMA20 → 开多;≤ -1.5 且价格跌破 EMA20 → 开空。价格过滤避免"大户接刀我跟接"。
- **出场**:背离 z 差收敛到 ±0.5 内平仓;止损 1.5×ATR。
- **max_hold**:24h(本库持仓最长的策略)
- **因子**:`LSR_TOP_POS` `LSR_TOP_ACC` `LSR_GLOBAL` `TECH_MA_EMA_BOLL_MACD_ATR`
- **风险点**:比率数据是所级统计,单所可能失真;可加 `LSR_HL`(Hyperliquid 链上人群)做第二确认。

## S05 · OI_TREND_RIDE — OI 确认趋势跟踪

**类型**:趋势跟踪 | **方向**:双向 | **适用**:日内单边行情

**逻辑**:健康趋势 = 价格新高 + 新钱入场(OI 上升)+ 主动买盘推动;三者齐备才追,缺一不追,OI 掉头即离场。

- **入场**(5m 扫描):1h K 线收盘创 12 根新高,且 `OI_BINANCE` 1h 增幅 ≥ +2% ⚙,且 `CVD` 同步创新高,且 `PX_BASIS` 基差未极端(<0.15%,防杠杆过热末段)→ 开多;镜像开空。
- **出场**:`OI_BINANCE` 1h 下降 ≥ 1.5% 或 CVD 与价格背离 → 平仓;否则 1.5×ATR 移动止损吊灯离场。
- **max_hold**:20h
- **因子**:`OI_BINANCE` `OI_AGG` `CVD` `PX_BASIS` `PX_FUT_KLINE` `TAKER_FLOW`
- **风险点**:震荡市来回止损;用 4h ATR/价格 比值 < 1.2% 时自动停用(波动不足)。

## S06 · CVD_FADE — CVD 背离反转

**类型**:微观结构反转 | **方向**:双向 | **适用**:冲高/杀跌末端

**逻辑**:价格靠稀薄挂单打出新高但主动买量(CVD)不创新高 = 推手是挂单撤退而非真实买盘,背离后大概率回摆。

- **入场**(15m 扫描):价格创 4h 新高但 `CVD`(现货+合约双确认)低于其前高 ≥ 15% 幅度 ⚙,且 `OB_DEPTH` 卖压比 ≥ 1.3,且 `OB_WALL` 上方 1% 内存在大额卖墙 → 开空;镜像开多。
- **出场**:回归 1h VWAP 止盈;新高上破卖墙价位止损(墙被吃掉 = 逻辑失效)。
- **max_hold**:6h
- **因子**:`CVD` `OB_DEPTH` `OB_WALL` `PX_SPOT_KLINE` `PX_FUT_KLINE`
- **风险点**:背离可以持续背离;单笔风险减半(0.5×标准仓),靠高胜率小止损盈利。

## S07 · EXCH_FLOW_ALERT — 交易所大额资金流事件

**类型**:链上事件驱动 | **方向**:双向 | **适用**:全时段(事件触发)

**逻辑**:大额充值进交易所 = 潜在抛售弹药;大额提走 = 囤币减压。事件后 1-6 小时窗口内价格有统计漂移。

- **入场**(15m 扫描):`EXCH_CHAIN_TX` 单笔 ≥ 5000 万 U ⚙ 流入币安/OKX(且 `WHALE_TX` 交叉确认非交易所间内部划转、非冷热钱包整理——同一实体标签对倒的剔除)→ 对该币开空,反向流出 → 开多。叠加条件:该币 `EXCH_BALANCE` 当日趋势同向。
- **出场**:固定 6h 时间出场 ⚙ 或 ±1.2×ATR 止盈止损先到先出。
- **max_hold**:6h
- **因子**:`EXCH_CHAIN_TX` `WHALE_TX` `EXCH_BALANCE` `HL_WHALE`
- **风险点**:标签误判(做市商日常调仓);单事件仓位 ≤ 标准仓 0.7 倍,同币 24h 内不重复触发。

## S08 · CB_PREMIUM_US — Coinbase 溢价美盘时段

**类型**:时段性资金流 | **方向**:双向 | **适用**:美盘(北京时间 21:30–05:00)

**逻辑**:Coinbase 溢价是美国合规资金(机构/ETF 申赎盘)净买压的实时代理;美盘时段溢价的方向与斜率对 BTC/ETH 短线有引导性。

- **入场**(15m 扫描,仅美盘时段激活):`COINBASE_PREM` 由负转正且 30m 斜率 > 0,且前一交易日 `ETF_FLOW_BTC` 为净流入(隔日确认的背景过滤)→ 开多 BTC/ETH;溢价由正转负且斜率 < 0 → 开空。
- **出场**:美盘结束(05:00)无条件平仓;时段内溢价回到 0 轴平仓;止损 1×ATR。
- **max_hold**:7.5h(时段长度)
- **因子**:`COINBASE_PREM` `ETF_FLOW_BTC` `PX_SPOT_KLINE`
- **风险点**:仅适用 BTC/ETH 等 ETF 标的;周末美股休市自动停用。

## S09 · EVENT_BREAKOUT — 宏观数据突破

**类型**:事件驱动动量 | **方向**:双向 | **适用**:CPI/FOMC/非农等定时数据

**逻辑**:定时宏观数据落地瞬间的首个有效方向,配合成交量与主动盘确认后追入,吃事件重定价的第一波。事前策略静默由全局风控层负责,本策略只做事后。

- **入场**(事件时刻起 1m 高频扫描 30 分钟):数据公布后首根 1m K 线方向确立,等第 2-3 根不回补且 `TAKER_FLOW`(1m 粒度,binance 官方)同向 ≥ 1.6 ⚙、成交量 ≥ 20 均量 3 倍 → 顺方向追入 BTC。
- **出场**:2×ATR 止盈或 0.8×ATR 止损 ⚙;事件后 3h 无论盈亏平仓(重定价完成)。
- **max_hold**:3h
- **因子**:`ECON_CAL` `PX_FUT_KLINE` `TAKER_FLOW` `AGG_TRADES`
- **风险点**:插针双杀;入场滑点用限价追单(限价 = 现价 ± 0.05%),3 秒未成交撤单放弃。

## S10 · XSEC_ROTATE — 横截面强弱轮动

**类型**:横截面动量 | **方向**:多空对冲 | **适用**:山寨活跃期(`ALT_SEASON` > 50 时效果最佳)

**逻辑**:日内资金在币种间轮动;用横截面打分选最强做多、最弱做空,市场中性对冲大盘方向,吃相对强弱差。

- **调仓**(每 4h 一次 ⚙):对流动性 Top50 合约币打分 = 0.4×(4h 涨跌幅 z)+ 0.3×(`NETFLOW_XSEC` 净流入 z)+ 0.2×(`TAKER_FLOW` z)+ 0.1×(`OI_AGG` 增速 z)。做多 Top3、做空 Bottom3 ⚙,等名义对冲。剔除:24h 内有解锁事件的币(`UNLOCK`)、费率极端币(留给 S01)。
- **出场**:下一次调仓整体换仓;单腿 -2×ATR 硬止损;全组合日内 -1.5% 熔断。
- **max_hold**:24h(每日 UTC 0 点全平重置)
- **因子**:`MKT_PRICE_CHG` `MKT_FUT_SNAP` `NETFLOW_XSEC` `TAKER_FLOW` `OI_AGG` `UNLOCK` `ALT_SEASON` `TECH_RSI_XSEC`
- **风险点**:多空腿同涨同跌的相关性踩踏;两腿必须同时成交,单腿失败立即撤另一腿。

## S11 · CRASH_SCALP — 高频接针(暴跌/暴涨清算捕捉)

**类型**:高频均值回归 | **方向**:双向 | **适用**:瞬时插针 / 清算瀑布(非趋势)

**逻辑**:暴跌/暴涨瞬间大量杠杆被强平,清算引擎的被迫单造成价格超调。用 WebSocket 逐笔价格做**秒级速度监控**,在刀势减缓(企稳)的一刻反向接针,吃清算后的均值回摆。快进快出,不追趋势。

- **入场**(**1 秒扫描**):观察窗口(默认 30s)内跌幅 ≥ 1.2% ⚙,且近 3s 刀势减缓(近端幅度 ≤ 窗口幅度 × 0.4)→ 市价接多;暴涨镜像接空。`liq_agg` 爆仓倍数因子可用时作为增强确认(默认不强制)。同币 300s 冷却防止一波反复进。
- **出场**:反弹/回落 0.6% ⚙ **自动止盈**(核心);继续恶化 0.8% 止损;浮盈过半且秒级动能反转 → 提前止盈;最长持仓 2h(实际多在数分钟内出场)。
- **max_hold**:2h(兜底,通常秒~分钟级平仓)
- **因子**:WebSocket 逐笔价格(秒级速度,真正的秒级信号)· `liq_agg`(爆仓确认)· 订单簿深度
- **秒级监控实现**:WebSocket bookTicker 逐笔价格写入滚动缓冲区,策略每 1 秒读缓冲区算窗口速度;仓位管理循环每 0.5 秒检查止盈——暴跌接针后价格反弹能在半秒内触发止盈。
- **风险点**:接飞刀本质高风险——严格小止损 + 秒级监控;黑天鹅单边下跌靠 0.8% 止损和全局日内熔断兜底,绝不加仓扛单。建议先小仓位模拟盘验证阈值(不同币波动差异大,`drop_pct` 可能需按币调)。

## 仓位与杠杆默认值(全策略共用基准 ⚙)

- 标准仓 = 单笔风险(止损距离)占净值 **0.5%**;由止损宽度反推名义仓位
- 杠杆上限 3x(S09 事件策略 2x)
- 同时最多持有 **6** 个币的仓位
- 所有市价单前先查 `OB_DEPTH` 估算滑点,预估滑点 > 0.15% 改限价或放弃
